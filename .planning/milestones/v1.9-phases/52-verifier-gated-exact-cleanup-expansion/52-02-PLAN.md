<risks>
- Adding repair settings to `OptimizerBudget.as_dict()` would churn historical run ids; mitigate by keeping repair settings separate and optional.
- A focused suite can overclaim if it assumes improvement; mitigate by setting `expect_recovery=False` and documenting measured before/after results in 52-03.
- Campaign columns can blur taxonomy if repaired rows are counted as blind or same-AST; mitigate with tests on `classify_run()`/`evidence_class_for_payload()` and campaign outputs.
- Perturbed-tree fallback can regress if expanded target-free cleanup absorbs all failures; mitigate by preserving target-aware repair as the second path when target-free cleanup fails.
</risks>
