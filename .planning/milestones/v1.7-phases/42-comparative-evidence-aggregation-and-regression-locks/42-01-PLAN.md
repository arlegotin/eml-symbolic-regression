# Phase 42 Plan: Comparative Evidence Aggregation and Regression Locks

**Status:** Ready
**Requirements:** EVD-03, EVD-04, EVD-05

## Goal

Users can compare raw EML and centered-family behavior through aggregate evidence instead of individual run inspection.

## Tasks

1. Extend benchmark run metrics with centered anomaly counters and formula-size proxies used by family diagnostics.
2. Add campaign table outputs for operator-family recovery grouped by formula, regime, depth, family, and schedule.
3. Add campaign diagnostic outputs for anomaly totals, verifier pass rates, repair/refit usage, unsupported rates, and formula active-node overhead.
4. Add a compact JSON lock for operator-family aggregate counts to support regression tests and future paper decision automation.
5. Add tests that run a filtered family campaign and assert the comparison/diagnostic/lock outputs preserve raw defaults and family metadata.

## Success Criteria

1. Family campaign tables report recovery rates by operator family, formula, start mode, training mode, schedule, and depth.
2. Diagnostics report anomaly totals, verifier pass rates, repair/refit usage, unsupported rate, and active-node overhead.
3. Existing campaign reports link the new family comparison artifacts.
4. Tests cover the new outputs and prove non-family smoke reporting still works.
