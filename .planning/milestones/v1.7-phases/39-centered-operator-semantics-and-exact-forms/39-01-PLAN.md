# Phase 39 Plan: Centered Operator Semantics and Exact Forms

**Status:** Ready
**Requirements:** OPF-01, OPF-02, OPF-03, OPF-04, OPF-05

## Goal

Users can represent, evaluate, serialize, and verify centered/scaled EML-family nodes without disturbing raw EML semantics.

## Tasks

1. Add operator-family metadata and centered-family semantics helpers in `semantics.py`.
2. Add centered exact AST support in `expression.py`, including NumPy, Torch, mpmath, SymPy, JSON round-trip, constant traversal, and safe document metadata.
3. Update raw/centered expression comparison helpers in `master_tree.py` without changing raw defaults.
4. Add focused tests covering centered semantics, AST round-trip, mpmath/SymPy evaluation, and shifted-singularity diagnostics.
5. Run the focused test set and record verification.

## Success Criteria

1. Semantics APIs support raw EML, `cEML_{s,t}`, `CEML_s`, and `ZEML_s` with explicit training and verification modes.
2. Centered-family implementations use `expm1` and `log1p`, preserve PyTorch `complex128` defaults, and keep training guards separate from faithful verification.
3. Exact AST JSON records operator family, `s`, `t`, terminal convention, and children, and round-trips deterministically.
4. SymPy and mpmath evaluation paths cover centered-family exact ASTs and are tested against backend semantics.
5. Anomaly diagnostics report non-finites, overflow pressure, branch/domain events, and shifted-singularity distance for centered nodes.
