## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| suite metadata -> benchmark execution | Built-in or JSON cases become executable training jobs and proof rows. |
| target inventory -> proof claim | Synthetic exact target IDs can inflate the perturbed-basin claim if not declared and validated. |
| perturbed runner -> aggregate evidence | Top-level status and evidence class determine bounded threshold pass/fail. |
| CLI filters -> execution | User-supplied `--start-mode` and filter values select runner paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01-01 | Tampering | `BenchmarkCase.validate` | mitigate | Validate `perturbed_tree` proof cases require `training_mode == 'perturbed_true_tree_training'`, nonzero perturbation, bounded threshold, and declared case IDs. |
| T-31-01-02 | Repudiation | `basin.py` perturbed manifests | mitigate | Serialize `return_kind`, target AST, perturbation config, active slot changes, verifier report, and initializer `kind == 'perturbed_true_tree'`. |
| T-31-01-03 | Spoofing | `evidence_class_for_payload` | mitigate | Derive `perturbed_true_tree_recovered` only from the new training mode and nonzero perturbation; keep compiler warm-start same-AST mapped to `same_ast`. |
| T-31-01-04 | Denial of Service | `proof-perturbed-basin` suite | mitigate | Keep points, seeds, depths, restarts, and warm steps fixed and small; reject negative seeds/noise through existing validation. |
| T-31-01-05 | Information Disclosure | proof dataset manifests | accept | Manifests expose formulas, domains, hashes, and provenance only; no secrets, raw arrays, or external user data are introduced. |
</threat_model>
