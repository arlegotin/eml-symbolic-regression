| Boundary | Description |
|----------|-------------|
| aggregate JSON -> diagnostics report | Local aggregate files become bound evidence and claim recommendations. |
| probe rows -> proof narrative | High-noise rows outside the bounded suite can be misrepresented as proof successes or omitted. |
| CLI path arguments -> file reads/writes | User-supplied paths identify aggregate inputs and report output directories. |
| campaign report -> user interpretation | Proof thresholds and probe evidence must stay separate in human-readable output. |
