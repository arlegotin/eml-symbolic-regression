| scaffold initializer -> proof artifact | Initializer provenance can make blind recovery look stronger than it is if ambiguous. |
| optimizer result -> verifier evidence class | Training success must be derived from verifier output, not best loss alone. |

| T-30-06 | Repudiation | Optimizer manifest initialization logs | mitigate | Record scaffold kind, coefficient, variable, strategy, seed, and embedding diagnostics for every scaled-exp attempt. |
| T-30-07 | Elevation of Privilege | Benchmark evidence class derivation | mitigate | Leave `evidence_class_for_payload()` derived from status/training mode; tests assert filtered proof run is `blind_training_recovered`, not supplied by suite data. |
