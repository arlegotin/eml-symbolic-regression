<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> filesystem output | User-provided output paths create local JSON artifacts. |
| aggregate JSON -> campaign CSV/report | Proof metadata becomes human-readable campaign evidence. |
| report narrative -> user claim interpretation | Ambiguous wording can escalate measured or assisted evidence into unsupported training claims. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-03-01 | Tampering | `proof-dataset --output` | mitigate | Use existing `_write_json` parent creation only; no overwrite deletion, no shelling out, no path expansion beyond caller-provided local path. |
| T-29-03-02 | Repudiation | campaign manifest/tables | mitigate | Include claim IDs, evidence class, threshold policy, claim-level threshold summaries, dataset manifest digest, and provenance source/expression in CSV and manifest outputs. |
| T-29-03-03 | Elevation of Privilege | campaign report wording | mitigate | Add explicit sentence that bounded proof thresholds count only allowed verifier-owned training evidence classes and keep compile/catalog separate. |
| T-29-03-04 | Spoofing | `list-claims` output | accept | Output is read-only from the local code registry; malformed external suites are validated in Plan 02 before execution. |
</threat_model>
