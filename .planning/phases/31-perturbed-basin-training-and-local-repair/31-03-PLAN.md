<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| aggregate JSON -> diagnostics report | Local aggregate files become bound evidence and claim recommendations. |
| probe rows -> proof narrative | High-noise rows outside the bounded suite can be misrepresented as proof successes or omitted. |
| CLI path arguments -> file reads/writes | User-supplied paths identify aggregate inputs and report output directories. |
| campaign report -> user interpretation | Proof thresholds and probe evidence must stay separate in human-readable output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-03-01 | Tampering | `build_perturbed_basin_bound_report` | mitigate | Report computes support only from evidence classes and includes all Beer-Lambert rows with artifact paths and reasons. |
| T-31-03-02 | Repudiation | `basin-bound.json` / `basin-bound.md` | mitigate | Serialize raw row data, supported bound fields, unsupported noise values, and claim recommendation so narrowed bounds are auditable. |
| T-31-03-03 | Information Disclosure | CLI aggregate paths | accept | Inputs are local benchmark artifacts with formulas, metrics, and paths; no secrets or external data are introduced. |
| T-31-03-04 | Spoofing | campaign proof report | mitigate | Keep bound-probe suite rows outside threshold tables; report text names probe suite separately. |
| T-31-03-05 | Denial of Service | filtered benchmark smoke tests | mitigate | Run one bounded Beer row and one probe Beer row with existing CI-scale suite budgets; broader depth curves remain Phase 32. |
</threat_model>
